This documentation covers how to use the Broadcasts API, including listing, creating, and retrieving broadcast details.

## When to use this method

Use Broadcasts when you want to send a one-time message to multiple recipients without expecting a response. This method is ideal for marketing campaigns, announcements, or updates that don't require user interaction but you wish to get analytics from messages, kind of like an email newsletter but for WhatsApp. Remember that Broadcasts require the use of WhatsApp approved templates from your Hilos account.

## Create Broadcasts on your account

To create a broadcast on your account, use the following endpoint:

:::info
Check out the [API docs](https://hilos.io/docs/api#tag/Broadcast/operation/Create%20Broadcasts) of this process.
:::

**`POST`** `/api/broadcast`

`Authorization`: tokenAuth

Request parameters:

<table>
  <tr>
    <th>Data</th>
    <th>Data type</th>
    <th>Example</th>
  </tr>
  <tr>
    <th>name</th>
    <td>
      <pre>string</pre>
    </td>
    <td>
      <pre>my_broadcast</pre>
    </td>
  </tr>
  <tr>
    <th>start_on</th>
    <td>
      <pre>string</pre>
    </td>
    <td>
      <pre>1</pre>
    </td>
  </tr>
  <tr>
    <th>recipients</th>
    <td>
      <pre>array of objects</pre>
    </td>
    <td>
      <pre>[{`{},{},{}`}]</pre>
    </td>
  </tr>
  <tr>
    <th>whatsapp_template</th>
    <td>
      <pre>String</pre>
    </td>
    <td>
      <pre>df03e2cd-bc88-41d0-b5ab-7543e6065e0f</pre>

  </tr>
  <tr>
    <th>contact_tags</th>
    <td>
      <pre>array of objects</pre>
    </td>

:::info
**has_priority**: Use this in order to send a broadcast even if the contact has an open conversation in the Inbox (NEW or IN PROGRESS)
:::

## Get Broadcast Detail

To get the details of a specific broadcast, use the following endpoint:

:::info
Check out the [API docs](https://hilos.io/docs/api#tag/Broadcast/operation/Get%20Broadcast%20Detail) of this process.
:::

**`GET`** `/api/message-blast/{id}`

`Authorization`: tokenAuth

Path parameters:

- **`id`**: `integer` **(required)**

** Example response: **

```json
{
  "name": "string",
  "created_by": "string",
  ...
}
```
